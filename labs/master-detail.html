Lab: Master-Detail Pattern

The Master-Detail pattern splits the screen into two panels. The left master panel shows a scrollable list; clicking an item loads its full details into the right detail panel. Tabs inside the detail panel let you switch between views of the data.

Step 1 — HTML with Hardcoded Content

Write the full two-panel layout with real hardcoded data — no JavaScript. The left panel is a <ul> of items; the right panel shows one item's details with a tab bar and a table.

Open in a browser — you should see a list on the left and a detail card on the right. Clicking list items does nothing yet.

Step 3 — Dynamic Rendering with JavaScript

Remove the hardcoded <li> items and the detail content. Leave both sections empty and add a <script> tag before </body>.

Interactive idea: keep state in variables (currentId, currentTab), then re-render from that state after every click.

3a — Load data and render the list

let allPokemon = [];
let currentId  = null;
let currentTab = 'info';

async function loadPokemon() {
  // Load initial records for the master list.
  for (let i = 1; i <= 20; i++) {
    const res  = await fetch(`https://pokedextr.uwista.dev/pokemon/${i}`);
    const data = await res.json();
    allPokemon.push(data);
    renderList(); // show items as they arrive
  }
}

function renderList() {
  const ul = document.querySelector('#list');
  // Rebuild list from current state.
  let html = '';
  for (let p of allPokemon) {
    html += `<li class="${p.id === currentId ? 'selected' : ''}"
                onclick="selectPokemon(${p.id})">
      <img src="${p.image}" alt="${p.name}">
      ${p.name}
    </li>`;
  }
  ul.innerHTML = html;
}

3b — Show the detail panel

function selectPokemon(id) {
  // Update selected item and reset tab.
  currentId  = id;
  currentTab = 'info';
  // Re-render both panels to reflect state.
  renderList();
  renderDetail();
}

function renderDetail() {
  // Find selected object from source array.
  const pokemon = allPokemon.find(p => p.id === currentId);
  if (!pokemon) return;

  const detail = document.querySelector('#detail');

  let tabContent = ''; // table markup for active tab
  if (currentTab === 'info') {
    tabContent = `<table>
      <tr><th>ID</th>    <td>${pokemon.id}</td></tr>
      <tr><th>Name</th>  <td>${pokemon.name}</td></tr>
      <tr><th>Type 1</th><td>${pokemon.type1}</td></tr>
      <tr><th>Type 2</th><td>${pokemon.type2 || '-'}</td></tr>
    </table>`;
  } else {
    tabContent = `<table>
      <tr><th>HP</th>     <td>${pokemon.hp}</td></tr>
      <tr><th>Attack</th> <td>${pokemon.attack}</td></tr>
      <tr><th>Defense</th><td>${pokemon.defense}</td></tr>
      <tr><th>Speed</th>  <td>${pokemon.speed}</td></tr>
    </table>`;
  }

  detail.innerHTML = `
    <img src="${pokemon.image}" alt="${pokemon.name}">
    <h2>${pokemon.name}</h2>
    <div class="tabs">
      <button class="${currentTab === 'info'  ? 'active' : ''}" onclick="setTab('info')">Info</button>
      <button class="${currentTab === 'stats' ? 'active' : ''}" onclick="setTab('stats')">Stats</button>
    </div>
    ${tabContent}
  `;
}

function setTab(tab) {
  // Only tab changes here, selected item stays.
  currentTab = tab;
  renderDetail();
}

loadPokemon();

3c — Interactive flow (what runs on each click)

  • Click list item: selectPokemon(id) updates selected ID, then calls renderList() and renderDetail().
  • Click tab button: setTab(tab) updates only tab state, then calls renderDetail().
  • UI is always rebuilt from state variables, so list highlight and detail content stay in sync.
Clicking a Pokémon in the list now loads its detail. The Info and Stats tabs re-render the detail panel without a page reload.
