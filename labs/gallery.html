Lab: Gallery Pattern

In this lab you will build a Gallery page step by step. A gallery displays a collection of items as cards in a grid, with search and filter controls.

Step 1 — HTML with Hardcoded Content

Write the full page structure with real hardcoded cards. No JavaScript at all — just HTML. You need a <nav> with a search input, a .filters bar with filter buttons, and a #gallery div containing card elements.

<!DOCTYPE html>

</head>
<body>

  <nav>
    <a href="index.html">Home</a>
    <h1>Gallery</h1>
    <input type="text" id="search" placeholder="Search...">
  </nav>

  <div class="filters">
    <button class="active">All</button>

Step 3 — Dynamic Rendering with JavaScript

Remove the three hardcoded .card divs from #gallery (leave it empty). Add a <script> tag before </body> and drive all content from JavaScript.

Important pattern: do not filter inside the render function. Instead, build a new array first (filteredList) and pass it into renderGallery(filteredList).

3a — Load data from the API

let allPokemon = [];
let currentFilter = 'all';
let searchQuery = '';

async function loadPokemon() {
  // Fetch a small starter dataset.
  for (let i = 1; i <= 20; i++) {
    const response = await fetch(`https://pokedextr.uwista.dev/pokemon/${i}`);
    const data = await response.json();
    allPokemon.push(data);
    refreshGallery();   // re-render while data is loading
  }
}

loadPokemon();

3b — Build a new filtered array, then render

function renderGallery(list) {
  const gallery = document.querySelector('#gallery');

  // Convert data objects into card HTML.
  let html = '';
  for (let p of list) {
    html += `<div class="card">
      <img src="${p.image}" alt="${p.name}">
      <p><strong>${p.name}</strong></p>
      <p>${p.type1}${p.type2 ? ' / ' + p.type2 : ''}</p>
    </div>`;
  }

  gallery.innerHTML = html;
}

function getFilteredList() {
  // Start from the full source array.
  let filteredList = [...allPokemon];

  // 1) Search by name text.
  filteredList = filteredList.filter(p =>
    p.name.toLowerCase().includes(searchQuery.toLowerCase())
  );

  // 2) Filter by selected type.
  if (currentFilter !== 'all') {
    filteredList = filteredList.filter(p =>
      p.type1 === currentFilter || p.type2 === currentFilter
    );
  }

  return filteredList;
}

function refreshGallery() {
  // Derive data first, then render.
  const filteredList = getFilteredList();
  renderGallery(filteredList);
}

3c — Wire up search and filter buttons

function handleSearch(inputElement) {
  // Keep state in sync with input text.
  searchQuery = inputElement.value;
  refreshGallery();
}

function setFilter(type, buttonElement) {
  // Update filter state.
  currentFilter = type;
  // Clear old active styles.
  document.querySelectorAll('.filters button').forEach(btn => {
    btn.classList.remove('active');
  });
  // Mark the clicked button.
  buttonElement.classList.add('active');
  refreshGallery();
}

Update the filter buttons in the HTML to call setFilter():

<div class="filters">
  <button class="active" onclick="setFilter('all', this)">All</button>
  <button onclick="setFilter('fire', this)">Fire</button>
  <button onclick="setFilter('water', this)">Water</button>
  <button onclick="setFilter('grass', this)">Grass</button>
</div>

Also update the search input to bind directly in HTML:

<input type="text" id="search" placeholder="Search..." oninput="handleSearch(this)">

What is this here? In an inline onclick, this means the exact button that was clicked. So setFilter('fire', this) passes two things: the filter value ('fire') and the clicked button element. Inside setFilter, that second value is used as buttonElement.classList.add('active') so only the clicked button gets the active style.

3d — Interactive flow (what runs on each action)

  • User types in search: update searchQuery → call refreshGallery().
  • User clicks a filter button: update currentFilter and active button style → call refreshGallery().
  • refreshGallery() always does two steps: getFilteredList() then renderGallery(filteredList).
The gallery now loads from the API, filters by type, and filters by search text — all without a page reload.
