Lab: Carousel Pattern

The Carousel pattern shows one item at a time with Previous and Next buttons to navigate through the collection.

Step 1 — HTML with Hardcoded Content

Write the full carousel page with one hardcoded card and the three navigation buttons — no JavaScript. You need a centred <main>, a #slide div with a .card inside, and an <article> button bar below.

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>Carousel</title>
</head>
<body>

  <nav>
    <a href="index.html">Home</a>
    <h1>Carousel</h1>
  </nav>

  <main>

    <div id="slide">
      <div class="card">
        <img src="https://pokedextr.uwista.dev/images/25.png" alt="pikachu">
        <h2>Pikachu</h2>
        <p>#25 — Electric</p>
        <p>HP: 35 | Attack: 55 | Defense: 40</p>
      </div>
    </div>

    <article>
      <button id="prev-btn" disabled>Previous</button>
      <button id="random-btn">Random</button>
      <button id="next-btn">Next</button>
    </article>

  </main>

</body>
</html>
Open in a browser — you should see the Pikachu card centred on the page with three buttons below. Buttons do nothing yet.

Step 3 — Dynamic Rendering with JavaScript

Remove the hardcoded .card div from #slide (leave it empty). Add a <script> tag before </body>.

Interactive idea: store the current position in currentIndex; every navigation action updates that index and calls renderSlide().

3a — State and render function

let allPokemon   = [];
let currentIndex = 0;

function renderSlide() {
  // Read current record by index.
  const pokemon = allPokemon[currentIndex];
  if (!pokemon) return;

  // Rebuild slide markup from current record.
  document.querySelector('#slide').innerHTML = `
    <div class="card">
      <img src="${pokemon.image}" alt="${pokemon.name}">
      <h2>${pokemon.name}</h2>
      <p>#${pokemon.id} — ${pokemon.type1}${pokemon.type2 ? ' / ' + pokemon.type2 : ''}</p>
      <p>HP: ${pokemon.hp} | Attack: ${pokemon.attack} | Defense: ${pokemon.defense}</p>
    </div>
  `;

  // Disable buttons at list boundaries.
  document.querySelector('#prev-btn').disabled = currentIndex === 0;
  document.querySelector('#next-btn').disabled = currentIndex === allPokemon.length - 1;
}

3b — Navigation and events

function navigate(direction) {
  // direction: -1 for previous, +1 for next
  currentIndex += direction;
  renderSlide();
}

function goToRandom() {
  // Pick any valid index in the loaded array.
  currentIndex = Math.floor(Math.random() * allPokemon.length);
  renderSlide();
}

function handleKey(e) {
  // Keyboard shortcuts for the same navigation actions.
  if (e.key === 'ArrowLeft')  navigate(-1);
  if (e.key === 'ArrowRight') navigate(1);
}

Bind controls directly in HTML:

<body onkeydown="handleKey(event)">
  ...
  <button id="prev-btn" onclick="navigate(-1)">Previous</button>
  <button id="random-btn" onclick="goToRandom()">Random</button>
  <button id="next-btn" onclick="navigate(1)">Next</button>
</body>

3c — Load data

async function loadPokemon() {
  // Load all records before first render.
  for (let i = 1; i <= 20; i++) {
    const res  = await fetch(`https://pokedextr.uwista.dev/pokemon/${i}`);
    const data = await res.json();
    allPokemon.push(data);
  }
  renderSlide();   // call AFTER all data loaded so button disabled state is correct
}

loadPokemon();

3d — Interactive flow (what runs on each action)

  • Click Previous/Next (or press arrow keys): update currentIndex through navigate(), then re-render slide.
  • Click Random: set currentIndex to a random valid value, then re-render slide.
  • renderSlide() rebuilds the card HTML and updates button disabled states each time.
Important: Call renderSlide() only after all data is loaded. If you call it inside the loop, allPokemon.length - 1 keeps changing and the Next button may stay incorrectly disabled.
